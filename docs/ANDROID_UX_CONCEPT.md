# Android UX Concept

## UX Positioning

Android UX should feel like an inbox assistant, not a replacement messenger.

The app should help users understand and find messages, not pull them into a new mandatory messaging workflow.

## Core UX Surfaces

### Home

Primary surfaces:

- Important
- OTP
- Bank
- Cargo
- Billing
- Hospital
- Official
- Promotional
- Suspicious

### Search-first model

Search should be central:

- sender search
- message text search
- category chips
- saved queries
- recent searches

### Trust-first views

The app should prioritize:

- Important Messages
- Critical Security Messages
- Recent OTP and Access Codes
- Deliveries and Orders
- Bills and Utilities
- Appointments and Official Notices

### Clutter views

Secondary views:

- Promotional
- Likely Junk
- Suspicious

These should be framed carefully so users understand that suspicious classification is assistive, not perfect.

## Example Information Architecture

Top-level tabs or sections:

- Home
- Search
- Categories
- Review
- Preferences

Suggested quick chips:

- Bank
- OTP
- Cargo
- Billing
- Hospital
- Official
- Promotional
- Suspicious

## Trust Signals in UX

Each message card should be able to show:

- category
- confidence band
- short explanation

Examples:

- OTP/security wording detected
- cargo wording detected
- suspicious payment/link cues detected
- bahis/freebet language detected

## User-Controlled Organization

Android UX should favor user control:

- saved sender views
- saved category views
- user-defined hidden clutter views
- favorites for trusted senders
- optional personal categories

## UX Behaviors to Avoid

Avoid:

- forcing users to send or reply inside this app
- making the app feel like a message inbox replacement
- hiding critical messages automatically
- overclaiming certainty

## Recommended MVP UX

Android MVP UX should likely be:

- categorized search view
- important messages view
- suspicious/promotional review view
- sender/category chips
- message detail view with explanation

This is the safest route to user trust without becoming the primary SMS workflow.
